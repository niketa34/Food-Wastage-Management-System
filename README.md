# Food Wastage Management System üçé
A data-driven Food Wastage Management System built with Python, SQL, and Streamlit. It connects surplus food providers (restaurants, individuals) with NGOs/receivers for efficient redistribution, featuring CRUD operations  and data analysis of donation trends.
Problem: Food wastage is a major issue, happening alongside food insecurity.
Solution: An interactive platform to connect surplus food providers and receivers.
Key Value: Waste Reduction, Social Good, and Data-Driven Insights.
Streamlit Web Application: User-friendly interface for all interactions.
Core Functionality (CRUD): Add, view, update, and remove food listings.
Filtering: Search and filter listings by city, provider, food type, and meal type.
Data Analysis & Reporting: SQL queries visualized, showing trends in claims, availability, and provider contributions.
Tool/Technology	Category	Detail of Use in Project:
MySQL	Database System (RDBMS)	The open-source relational database used to store all food listings, provider, receivers and claims data.
PyMySQL	Python Library (Connector)	Python library used to establish the connection and execute SQL queries from Python/Streamlit code to the MySQL database.
Pandas	Python Library (Data Analysis)	Used to read data from CSV or SQL queries into a DataFrame for cleaning, processing, and preparing data for Streamlit visualizations.
Streamlit	Python Library (Web Framework)	The application framework used to build the interactive web UI for filtering, displaying data analysis, and performing CRUD operations.
